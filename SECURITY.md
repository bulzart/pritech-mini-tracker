# Security Policy

## Supported versions

This project is in active checkpoint-based development. Security fixes are
applied to the latest commit on the default branch.

| Version            | Supported |
| ------------------ | --------- |
| latest (`main`)    | ✅        |
| older commits/tags | ❌        |

## Reporting a vulnerability

Please report security issues **privately** — do not open a public issue.

- **Email:** bulzarti@gmail.com
- Include: a description, reproduction steps, affected routes/components, and
  the potential impact.

**Response targets:**

- Acknowledgement within **3 business days**.
- Initial assessment within **7 business days**.
- Coordinated disclosure once a fix is available.

A machine-readable contact is also published at
[`/.well-known/security.txt`](public/.well-known/security.txt) per RFC 9116.

## Handling

- Do not exploit a vulnerability beyond what is necessary to demonstrate it.
- Do not access, modify, or delete data that is not yours.
- We will credit reporters who follow this policy unless they prefer to remain
  anonymous.
